# 帮助中心

## 各个绘图模型有什么特色？

**Midjourney** - 艺术风格强，适合概念设计、插画创作，支持 U/V 操作和图片混合，通过提示词参数控制版本和模式

**DALL-E** - OpenAI 出品，支持尺寸、质量、风格参数，不支持垫图

**GPT-4o 绘图** - 基于 GPT-4o 的图像生成，支持尺寸、质量、背景透明度参数，复杂指令理解能力强

**GPT Image** - OpenAI 最新图像生成模型（gpt-image-1、gpt-image-1.5），质量高，指令跟随能力强

**Sora 绘图** - OpenAI 视频模型的图像版本，审查较严格

**Gemini 绘图** - Google 出品，支持多轮对话式图像编辑，垫图效果好

**Flux** - 开源图像生成模型，支持宽高比、负面提示词参数，仅 flux-kontext 系列支持垫图

**Grok 绘图** - xAI 出品，风格多样，响应快速，审查宽松

**豆包绘图** - 字节跳动出品，支持尺寸、种子、提示词相关度、水印等参数，中文理解能力强

**通义万相** - 阿里出品，中文提示词效果好

**Z-Image** - 完全无审查，支持 NSFW 内容生成

**抠抠图** - 这不是 AI 大模型，而是本站适配的一种专用图像处理 API，用于移除图片背景

## 什么模型支持输出透明通道/移除背景？

目前支持透明背景的模型：

- **GPT-4o 绘图** - 支持 `background` 参数设为 `transparent`，可直接生成透明背景图片，该模型会一定程度改变图像内容
- **抠抠图** - 专用图像处理 API，上传任意图片即可获得透明背景版本，不需要输入提示词，不会改变图像内容

其他模型暂不支持直接输出透明背景，如需透明背景建议先生成后使用抠抠图处理。

## 哪些是特殊生图模型？

**Midjourney** 是一个特殊的模型类型：

- 模型名称是虚拟的，实际通过提示词参数来选择不同的模式
- 可以在提示词中添加 `--fast`、`--relax`、`--turbo` 来控制生成速度
- 支持 `--v 5`、`--v 6` 等版本参数
- 支持 `--niji` 切换到二次元风格模型
- 通过 U1-U4 放大、V1-V4 变体等按钮操作来优化结果

**抠抠图** 是特殊的图像处理服务：
- 这不是 AI 大模型，而是本站适配的专用图像处理 API
- 不需要提示词，只需上传图片
- 自动识别并移除背景，输出透明 PNG

## 可以使用哪些 API？

本系统支持以下 API 格式，只要上游服务商兼容这些格式即可接入（官方或第三方均可）：

| API 格式 | 说明 | 适用模型 |
|---------|------|---------|
| **MJ-Proxy** | 兼容 midjourney-proxy 项目的 API | Midjourney |
| **DALL-E API** | OpenAI 图像生成 API 格式 | DALL-E、Flux、豆包、Z-Image |
| **OpenAI Chat** | OpenAI Chat Completions 格式 | GPT-4o 绘图、GPT Image、Sora 绘图、Grok 绘图、通义万相 |
| **Gemini API** | Google Gemini 原生 API | Gemini 绘图 |
| **抠抠图 API** | 专用抠图服务 API | 抠抠图 |
| **视频统一格式** | 统一的视频生成 API | 即梦、Veo、Sora 视频、Grok 视频 |

配置上游时，API 地址无需添加 `/v1` 后缀，系统会根据不同接口自动拼接正确的路径。
